Before working with the VCP, create a virtual environment with stable packages.

If you have Anaconda on a school computer or the laptop, do the following:

>Navigate to the VCP-dev directory 
	cd [YOUR USER]"OneDrive\Plasma Chamber\Common Software & Data\Chamber Software\VCP-dev>"
>Create new virtual environment if it doesn't exist  
	conda create -n vcp_gui python=3.10 -y
>Activate new or existing environment
	conda activate vcp_gui
>Install dependencies using pip
	If requirements.txt exists "pip install -r requirements.txt"
	Else, typical packages are pyinstaller, numpy, pyserial, matplotlib, pandas, scipy, slack-sdk
>Test your code changes before GUI compiling by just running the main launcher VCP.py
	python VCP.py
>Modify GUIversion.txt after changes are finalized.
>When complete, compile using the following command. The workpath can be changed, but just don't try to compile in a OneDrive managed folder
	pyinstaller --clean --noconfirm --workpath "C:\VCP_pyinstaller_work" --distpath "..\VCP-run" --icon=sail_logo.ico --version-file=GUIversion.txt --add-data "images;images" --onefile VCP.py
>To allow for filaments to be replaced without recompiling the GUI, the "filamentParams.pkl" file must be moved into wherever the GUI is being run, as it is not packaged within the exe itself.
